# 기억 정책 (Memory Policy)

## 개요

무엇을 기억하고, 무엇을 잊을지 정하는 정책입니다.
이 파일을 워크스페이스에 두면 캐릭터가 이 규칙에 따라 기억을 관리합니다.

## 기록 감도

세 가지 레벨 중 선택 (이 파일에서 설정):

### 관대 (Generous) ← 현재 설정
- 대부분의 대화 내용 기록
- 취향, 감정, 일상 대화 포함
- 기록하지 않는 것만 필터링

### 보통 (Normal)
- 의미 있는 대화만 기록
- 단순 인사, 잡담은 생략
- 결정, 요청, 감정 변화 위주

### 최소 (Minimal)
- 핵심 정보만 기록
- 이름, 관계, 중요 결정만
- 대부분의 대화는 기록 안 함

## 기록하는 것

- **취향/선호:** 좋아하는 음식, 음악, 게임 등
- **별명/호칭:** 서로 부르는 이름
- **관계:** 누가 누구의 친구/가족인지
- **에피소드:** 재밌었던 일, 감동적인 일, 중요한 사건
- **요청 사항:** "이거 기억해줘"라고 한 것
- **감정 변화:** 기뻐했던 것, 화났던 것, 슬펐던 것
- **습관/패턴:** 활동 시간대, 자주 하는 행동

## 기록하지 않는 것

- **비밀번호/토큰:** 절대 기록 금지
- **금융 정보:** 카드번호, 계좌번호 등
- **주소/전화번호:** 명시적 요청 없으면 기록 안 함
- **의료 정보:** 민감한 건강 정보
- **타인의 개인정보:** 제3자의 민감 정보

## 메모리 파일 구조

### Daily (memory/YYYY-MM-DD.md)
- 그날 있었던 일의 요약
- 대화 중 중요했던 포인트
- 감정, 분위기, 특이사항

### Long-term (MEMORY.md)
- Daily에서 추린 핵심 정보
- 사람별 정보, 관계도
- 장기적으로 알아야 할 사실

## 정리 주기

- **Daily 파일:** 7일 이상 된 파일은 핵심만 추려 MEMORY.md로 이관
- **MEMORY.md:** 월 1회 정리 — 오래되고 관련 없는 정보 제거
- **정리 시점:** 하트비트 중 여유 있을 때 자동으로

## 삭제 요청

- "이거 잊어줘" → 해당 기록 삭제
- "내 정보 다 지워줘" → 해당 유저 관련 기록 전부 삭제
- 주인의 삭제 요청은 즉시 이행
- 다른 유저의 삭제 요청은 해당 유저 본인 정보만 삭제

## 감도 변경

주인이 요청하면 감도 레벨을 변경할 수 있습니다:
- "기억 좀 더 많이 해줘" → 관대로 변경
- "필요한 것만 기억해" → 최소로 변경
