# 커스텀 명령어 시스템 (Custom Commands)

## 개요

특정 유저나 키워드에 자동으로 반응하는 시스템입니다.
주인(서버장)만 설정할 수 있으며, `MEMORY.md`의 `## Commands` 섹션에서 관리합니다.

## 명령어 유형

### 1. 유저 반응 (User Triggers)
특정 Discord 유저 ID가 메시지를 보내면 자동 반응.

```markdown
### User Triggers
- ID: 123456789 → 리액션: 👋 + 메시지: "어서와!"
- ID: 987654321 → 리액션: 🔥
```

### 2. 키워드 반응 (Keyword Triggers)
특정 키워드가 포함된 메시지에 자동 반응.

```markdown
### Keyword Triggers
- "좋은 아침" → 메시지: "좋은 아침이야!" + 리액션: ☀️
- "배고파" → 메시지: "뭐 먹을래? 추천해줄까?"
- "잘자" → 리액션: 🌙
```

### 3. 정규식 반응 (Pattern Triggers)
패턴 매칭으로 더 유연한 반응.

```markdown
### Pattern Triggers
- /^(ㅋ{3,}|ㅎ{3,})$/ → 리액션: 😂
- /고마워|감사/ → 리액션: 💕
```

## 반응 종류

- **리액션:** 이모지 리액션 추가
- **메시지:** 텍스트 메시지 전송
- **DM:** 다이렉트 메시지 전송 (주의해서 사용)

## 권한 시스템

- **주인만** 명령어 추가/수정/삭제 가능
- 주인 식별: `USER.md`에 등록된 이름 또는 Discord ID
- 다른 유저가 요청하면 정중히 거절: "그건 주인만 설정할 수 있어"

## 설정 방법

주인이 자연어로 요청:
- "김철수가 채팅하면 👋 리액션 달아줘"
- "'배고파' 키워드에 음식 추천해주는 거 추가해"
- "유저 반응 목록 보여줘"
- "123456789 반응 삭제해"

## MEMORY.md 기록 형식

```markdown
## Commands

### User Triggers
| Discord ID | Reaction | Message | Added |
|------------|----------|---------|-------|
| 123456789 | 👋 | 어서와! | 2025-03-15 |

### Keyword Triggers
| Keyword | Reaction | Message | Added |
|---------|----------|---------|-------|
| 좋은 아침 | ☀️ | 좋은 아침이야! | 2025-03-15 |
| 잘자 | 🌙 | - | 2025-03-16 |
```

## 주의사항

- 자동 반응이 과하면 스팸이 됨 — 같은 유저에게 연속 반응은 5분 간격
- 키워드 반응은 정확 매칭보다 포함 매칭 사용
- 민감한 키워드(욕설 등)에는 반응 설정 지양
- 명령어 충돌 시 유저 반응이 키워드 반응보다 우선
